325      progress bar to the given `file` (defaults to stdout) and will attempt 
326      to calculate remaining time and more.  By default, this progress bar 
327      will not be rendered if the file is not a terminal. 
328   
329      The context manager creates the progress bar.  When the context 
330      manager is entered the progress bar is already created.  With every 
331      iteration over the progress bar, the iterable passed to the bar is 
332      advanced and the bar is updated.  When the context manager exits, 
333      a newline is printed and the progress bar is finalized on screen. 
334

336      total progress can be expected to take at least several seconds. 
337      Because of this, the ProgressBar class object won't display 
338      progress that is considered too fast, and progress where the time 
339      between steps is less than a second. 
340

342      destroyed. 
343

345

352      iterating over the progress bar.  The update method accepts the number 
353      of steps to increment the bar with::
