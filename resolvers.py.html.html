64   <span class="s0">class </span><span class="s1">Criterion(object):</span>
65       <span class="s3">&quot;&quot;&quot;Representation of possible resolution results of a package. 
66    
67       This holds three attributes: 
68    
69       * `information` is a collection of `RequirementInformation` pairs. 
70         Each pair is a requirement contributing to this criterion, and the 
71         candidate that provides the requirement. 
72       * `incompatibilities` is a collection of all known not-to-work candidates 
73         to exclude from consideration. 
74       * `candidates` is a collection containing all possible candidates deducted 
75         from the union of contributing requirements and known incompatibilities. 
76         It should never be empty, except when the criterion is an attribute of a 
77         raised `RequirementsConflicted` (in which case it is always empty). 
78    
79       .. note:: 
80           This class is intended to be externally immutable. **Do not** mutate 
81           any of its attribute containers.

263      <span class="s0">def </span><span class="s1">_backtrack(self):</span>
264          <span class="s3">&quot;&quot;&quot;Perform backtracking. 
265   
266          When we enter here, the stack is like this:: 
267   
268              [ state Z ] 
269              [ state Y ] 
270              [ state X ] 
271              .... earlier states are irrelevant. 
272   
273          1. No pins worked for Z, so it does not have a pin. 
274          2. We want to reset state Y to unpinned, and pin another candidate. 
275          3. State X holds what state Y was before the pin, but does not 
276             have the incompatibility information gathered in state Y. 
277   
278          Each iteration of the loop will: 
279   
280          1.  Discard Z. 
281          2.  Discard Y but remember its incompatibility information gathered 
282              previously, and the failure we're dealing with right now. 
283          3.  Push a new state Y' based on X, and apply the incompatibility 
284              information from Y to Y'. 
285          4a. If this causes Y' to conflict, we need to backtrack again. Make Y' 
286              the new Z and go back to step 2. 
287          4b. If the incompatibilities apply cleanly, end backtracking. 
288          &quot;&quot;&quot;</span>

291              <span class="s0">del </span><span class="s1">self._states[-</span><span class="s5">1</span><span class="s1">]</span>
292  
293              <span class="s4"># Retrieve the last candidate pin and known incompatibilities.</span>

299              <span class="s1">]</span>
300  
301              <span class="s4"># Also mark the newly known incompatibility.</span>
302              <span class="s1">incompatibilities_from_broken.append((name</span><span class="s0">, </span><span class="s1">[candidate]))</span>
303  
304              <span class="s1">self._r.backtracking(candidate=candidate)</span>
305  
306              <span class="s4"># Create a new state from the last known-to-work one, and apply</span>
307              <span class="s4"># the previously gathered incompatibility information.</span>
