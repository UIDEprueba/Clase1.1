99   
100  <span class="s0">This package offer better performance than its counterpart Chardet. Here are some numbers.</span>
101  
102  <span class="s0">| Package       | Accuracy       | Mean per file (ms) | File per sec (est) |</span>
103  <span class="s0">| ------------- | :-------------: | :------------------: | :------------------: |</span>
104  <span class="s0">|      [chardet](https://github.com/chardet/chardet)        |     92 %     |     200 ms      |       5 file/sec        |</span>
105  <span class="s0">| charset-normalizer |    **98 %**     |     **39 ms**      |       26 file/sec    |</span>
106  
107  <span class="s0">| Package       | 99th percentile       | 95th percentile | 50th percentile |</span>
108  <span class="s0">| ------------- | :-------------: | :------------------: | :------------------: |</span>
109  <span class="s0">|      [chardet](https://github.com/chardet/chardet)        |     1200 ms     |     287 ms      |       23 ms        |</span>
110  <span class="s0">| charset-normalizer |    400 ms     |     200 ms      |       15 ms    |</span>
111  
112  <span class="s0">Chardet's performance on larger file (1MB+) are very poor. Expect huge difference on large payload.</span>
113  
114  <span class="s0">&gt; Stats are generated using 400+ files using default parameters. More details on used files, see GHA workflows.</span>
115  <span class="s0">&gt; And yes, these results might change at any time. The dataset can be updated to include more files.</span>
116  <span class="s0">&gt; The actual delays heavily depends on your CPU capabilities. The factors should remain the same.</span>
117  
118  <span class="s0">[cchardet](https://github.com/PyYoshi/cChardet) is a non-native (cpp binding) and unmaintained faster alternative with </span>
119  <span class="s0">a better accuracy than chardet but lower than this package. If speed is the most important factor, you should try it.</span>

236  <span class="s0">## 😇 Why</span>
237  
238  <span class="s0">When I started using Chardet, I noticed that it was not suited to my expectations, and I wanted to propose a</span>
239  <span class="s0">reliable alternative using a completely different method. Also! I never back down on a good challenge!</span>
240  
241  <span class="s0">I **don't care** about the **originating charset** encoding, because **two different tables** can</span>
242  <span class="s0">produce **two identical rendered string.**</span>
243  <span class="s0">What I want is to get readable text, the best I can. </span>
244  
245  <span class="s0">In a way, **I'm brute forcing text decoding.** How cool is that ? 😎</span>
246  
247  <span class="s0">Don't confuse package **ftfy** with charset-normalizer or chardet. ftfy goal is to repair unicode string whereas charset-normalizer to convert raw file in unknown encoding to unicode.</span>
248  
249  <span class="s0">## 🍰 How</span>
250  
251    <span class="s0">- Discard all charset encoding table that could not fit the binary content.</span>
252    <span class="s0">- Measure chaos, or the mess once opened (by chunks) with a corresponding charset encoding.</span>
253    <span class="s0">- Extract matches with the lowest mess detected.</span>
254    <span class="s0">- Additionally, we measure coherence / probe for a language.</span>
255  
256  <span class="s0">**Wait a minute**, what is chaos/mess and coherence according to **YOU ?**</span>
257  
258  <span class="s0">*Chaos :* I opened hundred of text files, **written by humans**, with the wrong encoding table. **I observed**, then</span>
259  <span class="s0">**I established** some ground rules about **what is obvious** when **it seems like** a mess.</span>
260   <span class="s0">I know that my interpretation of what is chaotic is very subjective, feel free to contribute in order to</span>
261   <span class="s0">improve or rewrite it.</span>
262  
263  <span class="s0">*Coherence :* For each language there is on earth, we have computed ranked letter appearance occurrences (the best we can). So I thought</span>
264  <span class="s0">that intel is worth something here. So I use those records against decoded text to check if I can detect intelligent design.</span>
265  
266  <span class="s0">## ⚡ Known limitations</span>
267  
268    <span class="s0">- Language detection is unreliable when text contains two or more languages sharing identical letters. (eg. HTML (english tags) + Turkish content (Sharing Latin characters))</span>
269    <span class="s0">- Every charset detector heavily depends on sufficient content. In common cases, do not bother run detection on very tiny content.</span>
