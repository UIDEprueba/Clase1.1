886  <span class="s0">class </span><span class="s1">LimitedStream(io.IOBase):</span>
887      <span class="s4">&quot;&quot;&quot;Wraps a stream so that it doesn't read more than n bytes.  If the 
888      stream is exhausted and the caller tries to get more bytes from it 
889      :func:`on_exhausted` is called which by default returns an empty 
890      string.  The return value of that function is forwarded 
891      to the reader function.  So if it returns an empty string 
892      :meth:`read` will return an empty string as well. 
893

895      output.  Otherwise :meth:`readlines` will try to read past the 
896      limit.

901         WSGI compliant because it passes a size argument to the 
902         readline methods.  Unfortunately the WSGI PEP is not safely 
903         implementable without a size argument to :meth:`readline` 
904         because there is no EOF marker in the stream.  As a result 
905         of that the use of :meth:`readline` is discouraged. 
906   
907         For the same reason iterating over the :class:`LimitedStream` 
908         is not portable.  It internally calls :meth:`readline`.

911         :func:`make_line_iter` which safely iterates line-based 
912         over a WSGI input stream.
