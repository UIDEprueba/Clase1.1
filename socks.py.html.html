21   <span class="s2">&quot;&quot;&quot; 
22   This module contains provisional support for SOCKS proxies from within 
23   urllib3. This module supports SOCKS4, SOCKS4A (an extension of SOCKS4), and 
24   SOCKS5. To enable its functionality, either install PySocks or install this 
25   module with the ``socks`` extra. 
26    
27   The SOCKS implementation supports the full range of urllib3 features. It also 
28   supports the following SOCKS features: 
29    
30   - SOCKS4A (``proxy_url='socks4a://...``) 
31   - SOCKS4 (``proxy_url='socks4://...``) 
32   - SOCKS5 with remote DNS (``proxy_url='socks5h://...``) 
33   - SOCKS5 with local DNS (``proxy_url='socks5://...``) 
34   - Usernames and passwords for the SOCKS proxy 
35    
36   .. note:: 
37      It is recommended to use ``socks5h://`` or ``socks4a://`` schemes in 
38      your ``proxy_url`` to ensure that DNS resolution is done from the remote 
39      server instead of client-side when connecting to a domain name. 
40    
41   SOCKS4 supports IPv4 and domain names with the SOCKS4A extension. SOCKS5 
42   supports IPv4, IPv6, and domain names. 
43    
44   When connecting to a SOCKS4 proxy the ``username`` portion of the ``proxy_url`` 
45   will be sent as the ``userid`` section of the SOCKS request: 
46    
47   .. code-block:: python 
48    
49       proxy_url=&quot;socks4a://&lt;userid&gt;@proxy-host&quot; 
50    
51   When connecting to a SOCKS5 proxy the ``username`` and ``password`` portion 
52   of the ``proxy_url`` will be sent as the username/password to authenticate 
53   with the proxy: 
54    
55   .. code-block:: python 
56    
57       proxy_url=&quot;socks5h://&lt;username&gt;:&lt;password&gt;@proxy-host&quot; 
58
