162  <span class="s2">class </span><span class="s1">Context:</span>
163      <span class="s0">&quot;&quot;&quot;The template context holds the variables of a template.  It stores the 
164      values passed to the template and also the names the template exports. 
165      Creating instances is neither supported nor useful as it's created 
166      automatically at various stages of the template evaluation and should not 
167      be created by hand. 
168   
169      The context is immutable.  Modifications on :attr:`parent` **must not** 
170      happen and modifications on :attr:`vars` are allowed from generated 
171      template code only.  Template filters and global functions marked as 
172      :func:`pass_context` get the active context passed as first argument 
173      and are allowed to access the context read-only. 
174   
175      The template context supports read only dict operations (`get`, 
176      `keys`, `values`, `items`, `iterkeys`, `itervalues`, `iteritems`, 
177      `__getitem__`, `__contains__`).  Additionally there is a :meth:`resolve` 
178      method that doesn't fail with a `KeyError` but returns an 
179      :class:`Undefined` object for missing variables. 
180      &quot;&quot;&quot;</span>
