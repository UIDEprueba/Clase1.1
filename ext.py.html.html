74       <span class="s0">&quot;&quot;&quot;Extensions can be used to add extra functionality to the Jinja template 
75       system at the parser level.  Custom extensions are bound to an environment 
76       but may not store environment specific data on `self`.  The reason for 
77       this is that an extension can be bound to another environment (for 
78       overlays) by creating a copy and reassigning the `environment` attribute. 
79    
80       As extensions are created by the environment they cannot accept any 
81       arguments for configuration.  One may want to work around that by using 
82       a factory function, but that is not possible as extensions are identified 
83       by their import name.  The correct way to configure the extension is 
84       storing the configuration values on the environment.  Because this way the 
85       environment ends up acting as central configuration storage the 
86       attributes may clash which is why extensions have to ensure that the names 
87       they choose for configuration are not too generic.  ``prefix`` for example 
88       is a terrible name, ``fragment_cache_prefix`` on the other hand is a good 
89       name as includes the name of the extension (fragment cache).
