20   <pre><span class="s0">&quot;&quot;&quot; 
21   SecureTranport support for urllib3 via ctypes. 
22    
23   This makes platform-native TLS available to urllib3 users on macOS without the 
24   use of a compiler. This is an important feature because the Python Package 
25   Index is moving to become a TLSv1.2-or-higher server, and the default OpenSSL 
26   that ships with macOS is not capable of doing TLSv1.2. The only way to resolve 
27   this is to give macOS users an alternative solution to the problem, and that 
28   solution is to use SecureTransport. 
29    
30   We use ctypes here because this solution must not require a compiler. That's 
31   because pip is not allowed to require a compiler either. 
32    
33   This is not intended to be a seriously long-term solution to this problem. 
34   The hope is that PEP 543 will eventually solve this issue for us, at which 
35   point we can retire this contrib module. But in the short term, we need to 
36   solve the impending tire fire that is Python on Mac without this kind of 
37   contrib module. So...here we are. 
38    
39   To use this module, simply import and inject it:: 
40    
41       import urllib3.contrib.securetransport 
42       urllib3.contrib.securetransport.inject_into_urllib3() 
43    
44   Happy TLSing!
