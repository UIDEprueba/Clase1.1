34   <span class="s2">class </span><span class="s1">CCompiler:</span>
35       <span class="s0">&quot;&quot;&quot;Abstract base class to define the interface that must be implemented 
36       by real compiler classes.  Also has some utility methods used by 
37       several compiler classes. 
38    
39       The basic idea behind a compiler abstraction class is that each 
40       instance can be used for all the compile/link steps in building a 
41       single project.  Thus, attributes common to all of those compile and 
42       link steps -- include directories, macros to define, libraries to link 
43       against, etc. -- are attributes of the compiler instance.  To allow for 
44       variability in how individual files are treated, most of those 
45       attributes may be varied on a per-compilation or per-link basis. 
46       &quot;&quot;&quot;</span>

51       <span class="s3"># 'isinstance'.  In concrete CCompiler subclasses, 'compiler_type'</span>

839      <span class="s3"># -- Filename generation methods -----------------------------------</span>
840  
841      <span class="s3"># The default implementation of the filename generating methods are</span>
842      <span class="s3"># prejudiced towards the Unix/DOS/Windows view of the world:</span>
843      <span class="s3">#   * object files are named by replacing the source file extension</span>
844      <span class="s3">#     (eg. .c/.cpp -&gt; .o/.obj)</span>
845      <span class="s3">#   * library files (shared or static) are named by plugging the</span>
846      <span class="s3">#     library name and extension into a format string, eg.</span>
847      <span class="s3">#     &quot;lib%s.%s&quot; % (lib_name, &quot;.a&quot;) for Unix static libraries</span>
848      <span class="s3">#   * executables are named by appending an extension (possibly</span>
849      <span class="s3">#     empty) to the program name: eg. progname + &quot;.exe&quot; for</span>
850      <span class="s3">#     Windows</span>
851      <span class="s3">#</span>
852      <span class="s3"># To reduce redundant code, these methods expect to find</span>
853      <span class="s3"># several attributes in the current object (presumably defined</span>
854      <span class="s3"># as class attributes):</span>
855      <span class="s3">#   * src_extensions -</span>
856      <span class="s3">#     list of C/C++ source file extensions, eg. ['.c', '.cpp']</span>
857      <span class="s3">#   * obj_extension -</span>
858      <span class="s3">#     object file extension, eg. '.o' or '.obj'</span>
859      <span class="s3">#   * static_lib_extension -</span>
860      <span class="s3">#     extension for static library files, eg. '.a' or '.lib'</span>
861      <span class="s3">#   * shared_lib_extension -</span>
862      <span class="s3">#     extension for shared library/object files, eg. '.so', '.dll'</span>
863      <span class="s3">#   * static_lib_format -</span>
864      <span class="s3">#     format string for generating static library filenames,</span>
865      <span class="s3">#     eg. 'lib%s.%s' or '%s.%s'</span>
866      <span class="s3">#   * shared_lib_format</span>
867      <span class="s3">#     format string for generating shared library filenames</span>
868      <span class="s3">#     (probably same as static_lib_format, since the extension</span>
869      <span class="s3">#     is one of the intended parameters to the format string)</span>
870      <span class="s3">#   * exe_extension -</span>
