34   <span class="s2">&quot;&quot;&quot; 
35   The ``distro`` package (``distro`` stands for Linux Distribution) provides 
36   information about the Linux distribution it runs on, such as a reliable 
37   machine-readable distro ID, or version information. 
38    
39   It is the recommended replacement for Python's original 
40   :py:func:`platform.linux_distribution` function, but it provides much more 
41   functionality. An alternative implementation became necessary because Python 
42   3.5 deprecated this function, and Python 3.8 removed it altogether. Its 
43   predecessor function :py:func:`platform.dist` was already deprecated since 
44   Python 2.6 and removed in Python 3.8. Still, there are many cases in which 
45   access to OS distribution information is needed. See `Python issue 1322 
46   &lt;https://bugs.python.org/issue1322&gt;`_ for more information. 
47   &quot;&quot;&quot;</span>

155  <span class="s3">def </span><span class="s1">linux_distribution(full_distribution_name=</span><span class="s3">True</span><span class="s1">):</span>
156      <span class="s0"># type: (bool) -&gt; Tuple[str, str, str]</span>
157      <span class="s2">&quot;&quot;&quot; 
158      .. deprecated:: 1.6.0 
159   
160          :func:`distro.linux_distribution()` is deprecated. It should only be 
161          used as a compatibility shim with Python's 
162          :py:func:`platform.linux_distribution()`. Please use :func:`distro.id`, 
163          :func:`distro.version` and :func:`distro.name` instead. 
164   
165      Return information about the current OS distribution as a tuple 
166      ``(id_name, version, codename)`` with items as follows: 
167   
168      * ``id_name``:  If *full_distribution_name* is false, the result of 
169        :func:`distro.id`. Otherwise, the result of :func:`distro.name`. 
170   
171      * ``version``:  The result of :func:`distro.version`. 
172   
173      * ``codename``:  The result of :func:`distro.codename`. 
174   
175      The interface of this function is compatible with the original 
176      :py:func:`platform.linux_distribution` function, supporting a subset of 
177      its parameters. 
178   
179      The data it returns may not exactly be the same, because it uses more data 
180      sources than the original function, and that may lead to different data if 
181      the OS distribution is not consistent across multiple data sources it 
182      provides (there are indeed such distributions ...). 
183   
184      Another reason for differences is the fact that the :func:`distro.id` 
185      method normalizes the distro ID string to a reliable machine-readable value 
186      for a number of popular OS distributions. 
187      &quot;&quot;&quot;</span>
188      <span class="s1">warnings.warn(</span>
189          <span class="s4">&quot;distro.linux_distribution() is deprecated. It should only be used as a &quot;</span>
190          <span class="s4">&quot;compatibility shim with Python's platform.linux_distribution(). Please use &quot;</span>
191          <span class="s4">&quot;distro.id(), distro.version() and distro.name() instead.&quot;</span><span class="s3">,</span>
192          <span class="s1">DeprecationWarning</span><span class="s3">,</span>
193          <span class="s1">stacklevel=</span><span class="s5">2</span><span class="s3">,</span>
194      <span class="s1">)</span>
195      <span class="s3">return </span><span class="s1">_distro.linux_distribution(full_distribution_name)</span>

198  <span class="s3">def </span><span class="s1">id():</span>
199      <span class="s0"># type: () -&gt; str</span>
200      <span class="s2">&quot;&quot;&quot; 
201      Return the distro ID of the current distribution, as a 
202      machine-readable string. 
203   
204      For a number of OS distributions, the returned distro ID value is 
205      *reliable*, in the sense that it is documented and that it does not change 
206      across releases of the distribution. 
207   
208      This package maintains the following reliable distro ID values: 
209   
210      ==============  ========================================= 
211      Distro ID       Distribution 
212      ==============  ========================================= 
213      &quot;ubuntu&quot;        Ubuntu 
214      &quot;debian&quot;        Debian 
215      &quot;rhel&quot;          RedHat Enterprise Linux 
216      &quot;centos&quot;        CentOS 
217      &quot;fedora&quot;        Fedora 
218      &quot;sles&quot;          SUSE Linux Enterprise Server 
219      &quot;opensuse&quot;      openSUSE 
220      &quot;amazon&quot;        Amazon Linux 
221      &quot;arch&quot;          Arch Linux 
222      &quot;cloudlinux&quot;    CloudLinux OS 
223      &quot;exherbo&quot;       Exherbo Linux 
224      &quot;gentoo&quot;        GenToo Linux 
225      &quot;ibm_powerkvm&quot;  IBM PowerKVM 
226      &quot;kvmibm&quot;        KVM for IBM z Systems 
227      &quot;linuxmint&quot;     Linux Mint 
228      &quot;mageia&quot;        Mageia 
229      &quot;mandriva&quot;      Mandriva Linux 
230      &quot;parallels&quot;     Parallels 
231      &quot;pidora&quot;        Pidora 
232      &quot;raspbian&quot;      Raspbian 
233      &quot;oracle&quot;        Oracle Linux (and Oracle Enterprise Linux) 
234      &quot;scientific&quot;    Scientific Linux 
235      &quot;slackware&quot;     Slackware 
236      &quot;xenserver&quot;     XenServer 
237      &quot;openbsd&quot;       OpenBSD 
238      &quot;netbsd&quot;        NetBSD 
239      &quot;freebsd&quot;       FreeBSD 
240      &quot;midnightbsd&quot;   MidnightBSD 
241      ==============  ========================================= 
242   
243      If you have a need to get distros for reliable IDs added into this set, 
244      or if you find that the :func:`distro.id` function returns a different 
245      distro ID for one of the listed distros, please create an issue in the 
246      `distro issue tracker`_. 
247   
248      **Lookup hierarchy and transformations:** 
249   
250      First, the ID is obtained from the following sources, in the specified 
251      order. The first available and non-empty value is used: 
252   
253      * the value of the &quot;ID&quot; attribute of the os-release file, 
254   
255      * the value of the &quot;Distributor ID&quot; attribute returned by the lsb_release 
256        command, 
257   
258      * the first part of the file name of the distro release file, 
259   
260      The so determined ID value then passes the following transformations, 
261      before it is returned by this method: 
262   
263      * it is translated to lower case, 
264   
265      * blanks (which should not be there anyway) are translated to underscores, 
266   
267      * a normalization of the ID is performed, based upon 
268        `normalization tables`_. The purpose of this normalization is to ensure 
269        that the ID is as reliable as possible, even across incompatible changes 
270        in the OS distributions. A common reason for an incompatible change is 
271        the addition of an os-release file, or the addition of the lsb_release 
272        command, with ID values that differ from what was previously determined 
273        from the distro release file name. 
274      &quot;&quot;&quot;</span>
275      <span class="s3">return </span><span class="s1">_distro.id()</span>

280      <span class="s2">&quot;&quot;&quot; 
281      Return the name of the current OS distribution, as a human-readable 
282      string. 
283   
284      If *pretty* is false, the name is returned without version or codename. 
285      (e.g. &quot;CentOS Linux&quot;) 
286   
287      If *pretty* is true, the version and codename are appended. 
288      (e.g. &quot;CentOS Linux 7.1.1503 (Core)&quot;) 
289   
290      **Lookup hierarchy:** 
291   
292      The name is obtained from the following sources, in the specified order. 
293      The first available and non-empty value is used: 
294   
295      * If *pretty* is false: 
296   
297        - the value of the &quot;NAME&quot; attribute of the os-release file, 
298   
299        - the value of the &quot;Distributor ID&quot; attribute returned by the lsb_release 
300          command, 
301   
302        - the value of the &quot;&lt;name&gt;&quot; field of the distro release file. 
303   
304      * If *pretty* is true: 
305   
306        - the value of the &quot;PRETTY_NAME&quot; attribute of the os-release file, 
307   
308        - the value of the &quot;Description&quot; attribute returned by the lsb_release 
309          command, 
310   
311        - the value of the &quot;&lt;name&gt;&quot; field of the distro release file, appended 
312          with the value of the pretty version (&quot;&lt;version_id&gt;&quot; and &quot;&lt;codename&gt;&quot; 
313          fields) of the distro release file, if available. 
314      &quot;&quot;&quot;</span>

318  <span class="s3">def </span><span class="s1">version(pretty=</span><span class="s3">False, </span><span class="s1">best=</span><span class="s3">False</span><span class="s1">):</span>
319      <span class="s0"># type: (bool, bool) -&gt; str</span>
320      <span class="s2">&quot;&quot;&quot; 
321      Return the version of the current OS distribution, as a human-readable 
322      string. 
323   
324      If *pretty* is false, the version is returned without codename (e.g. 
325      &quot;7.0&quot;). 
326   
327      If *pretty* is true, the codename in parenthesis is appended, if the 
328      codename is non-empty (e.g. &quot;7.0 (Maipo)&quot;). 
329   
330      Some distributions provide version numbers with different precisions in 
331      the different sources of distribution information. Examining the different 
332      sources in a fixed priority order does not always yield the most precise 
333      version (e.g. for Debian 8.2, or CentOS 7.1). 
334   
335      The *best* parameter can be used to control the approach for the returned 
336      version: 
337   
338      If *best* is false, the first non-empty version number in priority order of 
339      the examined sources is returned. 
340   
341      If *best* is true, the most precise version number out of all examined 
342      sources is returned. 
343   
344      **Lookup hierarchy:** 
345   
346      In all cases, the version number is obtained from the following sources. 
347      If *best* is false, this order represents the priority order: 
348   
349      * the value of the &quot;VERSION_ID&quot; attribute of the os-release file, 
350      * the value of the &quot;Release&quot; attribute returned by the lsb_release 
351        command, 
352      * the version number parsed from the &quot;&lt;version_id&gt;&quot; field of the first line 
353        of the distro release file, 
354      * the version number parsed from the &quot;PRETTY_NAME&quot; attribute of the 
355        os-release file, if it follows the format of the distro release files. 
356      * the version number parsed from the &quot;Description&quot; attribute returned by 
357        the lsb_release command, if it follows the format of the distro release 
358        files. 
359      &quot;&quot;&quot;</span>
360      <span class="s3">return </span><span class="s1">_distro.version(pretty</span><span class="s3">, </span><span class="s1">best)</span>
