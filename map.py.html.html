515          <span class="s2">&quot;&quot;&quot;The usage is simple: you just pass the match method the current 
516          path info as well as the method (which defaults to `GET`).  The 
517          following things can then happen: 
518   
519          - you receive a `NotFound` exception that indicates that no URL is 
520            matching.  A `NotFound` exception is also a WSGI application you 
521            can call to get a default page not found page (happens to be the 
522            same object as `werkzeug.exceptions.NotFound`) 
523   
524          - you receive a `MethodNotAllowed` exception that indicates that there 
525            is a match for this URL but not for the current request method. 
526            This is useful for RESTful applications. 
527   
528          - you receive a `RequestRedirect` exception with a `new_url` 
529            attribute.  This exception is used to notify you about a request 
530            Werkzeug requests from your WSGI application.  This is for example the 
531            case if you request ``/foo`` although the correct URL is ``/foo/``

533            similar to all other subclasses of `HTTPException`. 
534

536            match is a WebSocket rule but the bind is an HTTP request, or 
537            if the match is an HTTP rule but the bind is a WebSocket 
538            request. 
539   
540          - you get a tuple in the form ``(endpoint, arguments)`` if there is 
541            a match (unless `return_rule` is True, in which case you get a tuple 
542            in the form ``(rule, arguments)``) 
543

545          info of the map is used (defaults to the root URL if not defined 
546          explicitly). 
547

549          can be used as WSGI responses. They will all render generic error or 
550          redirect pages.
