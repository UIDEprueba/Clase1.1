52   <span class="s2">class </span><span class="s1">CodingStateMachine(object):</span>
53       <span class="s3">&quot;&quot;&quot; 
54       A state machine to verify a byte sequence for a particular encoding. For 
55       each byte the detector receives, it will feed that byte to every active 
56       state machine available, one byte at a time. The state machine changes its 
57       state based on its previous state and the byte it receives. There are 3 
58       states in a state machine that are of interest to an auto-detector: 
59    
60       START state: This is the state to start with, or a legal byte sequence 
61                    (i.e. a valid code point) for character has been identified. 
62    
63       ME state:  This indicates that the state machine identified a byte sequence 
64                  that is specific to the charset it is designed for and that 
65                  there is no other possible encoding which can contain this byte 
66                  sequence. This will to lead to an immediate positive answer for 
67                  the detector. 
68    
69       ERROR state: This indicates the state machine identified an illegal byte 
70                    sequence for that encoding. This will lead to an immediate 
71                    negative answer for this encoding. Detector will exclude this 
72                    encoding from consideration from here on. 
73       &quot;&quot;&quot;</span>
