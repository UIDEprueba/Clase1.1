112

114      the installer filename and the name of the egg-info directory embedded in 
115      the zipfile (if any). 
116   
117      The egginfo filename has the format:: 
118   
119          name-ver(-pyver)(-arch).egg-info 
120   
121      The installer filename has the format:: 
122   
123          name-ver.arch(-pyver).exe 
124   
125      Some things to note: 
126   
127      1. The installer filename is not definitive. An installer can be renamed 
128         and work perfectly well as an installer. So more reliable data should 
129         be used whenever possible. 
130      2. The egg-info data should be preferred for the name and version, because 
131         these come straight from the distutils metadata, and are mandatory. 
132      3. The pyver from the egg-info data should be ignored, as it is 
133         constructed from the version of Python used to build the installer, 
134         which is irrelevant - the installer filename is correct here (even to 
135         the point that when it's not there, any version is implied). 
136      4. The architecture must be taken from the installer filename, as it is 
137         not included in the egg-info data. 
138      5. Architecture-neutral installers still have an architecture because the 
139         installer format itself (being executable) is architecture-specific. We 
140         should therefore ignore the architecture if the content is pure-python.
