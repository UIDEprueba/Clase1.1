45   <span class="s0">class </span><span class="s1">Serializer:</span>
46       <span class="s2">&quot;&quot;&quot;A serializer wraps a :class:`~itsdangerous.signer.Signer` to 
47       enable serializing and securely signing data other than bytes. It 
48       can unsign to verify that the data hasn't been changed. 
49    
50       The serializer provides :meth:`dumps` and :meth:`loads`, similar to 
51       :mod:`json`, and by default uses :mod:`json` internally to serialize 
52       the data to bytes. 
53    
54       The secret key should be a random string of ``bytes`` and should not 
55       be saved to code or version control. Different salts should be used 
56       to distinguish signing in different contexts. See :doc:`/concepts` 
57       for information about the security of the secret key and salt. 
58    
59       :param secret_key: The secret key to sign and verify with. Can be a 
60           list of keys, oldest to newest, to support key rotation. 
61       :param salt: Extra key to combine with ``secret_key`` to distinguish 
62           signatures in different contexts. 
63       :param serializer: An object that provides ``dumps`` and ``loads`` 
64           methods for serializing data to a string. Defaults to 
65           :attr:`default_serializer`, which defaults to :mod:`json`. 
66       :param serializer_kwargs: Keyword arguments to pass when calling 
67           ``serializer.dumps``. 
68       :param signer: A ``Signer`` class to instantiate when signing data. 
69           Defaults to :attr:`default_signer`, which defaults to 
70           :class:`~itsdangerous.signer.Signer`. 
71       :param signer_kwargs: Keyword arguments to pass when instantiating 
72           the ``Signer`` class. 
73       :param fallback_signers: List of signer parameters to try when 
74           unsigning with the default signer fails. Each item can be a dict 
75           of ``signer_kwargs``, a ``Signer`` class, or a tuple of 
76           ``(signer, signer_kwargs)``. Defaults to 
77           :attr:`default_fallback_signers`. 
78    
79       .. versionchanged:: 2.0 
80           Added support for key rotation by passing a list to 
81           ``secret_key``. 
82    
83       .. versionchanged:: 2.0 
84           Removed the default SHA-512 fallback signer from 
85           ``default_fallback_signers``. 
86    
87       .. versionchanged:: 1.1 
88           Added support for ``fallback_signers`` and configured a default 
89           SHA-512 fallback. This fallback is for users who used the yanked 
90           1.0.0 release which defaulted to SHA-512. 
91    
92       .. versionchanged:: 0.14 
93           The ``signer`` and ``signer_kwargs`` parameters were added to 
94           the constructor.
